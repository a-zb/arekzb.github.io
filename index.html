Search on new PHP.net is not Firefox friendly

Written by arekzb on July 22, 2013.

I like the new design of PHP.net website. It sometimes happens in a beta phase,- there's an issue that in my point of view is a show stopper.

I have tried to use the new search widget on php.net and have found it to be a rather obtuse piece to work with. Not only does it enforce focus by taking the value of the element present under the mouse, but it will happily deny you using the backspace key to clear up the wrong suggestion. The mouse does not have to be moved at all. It simply needs to be above an element in the list.

Chrome doesn't suffer suffer from the same behaviour, but it is not a browser bug. Developers at php.net should take out the focus attribute, passed to catcomplete in common.js script file. 

                $('#headsearch-keywords').catcomplete({
                ...
                focus: function(event, ui) {
                    $('#headsearch-keywords').val(ui.item.label);
                    return false;
                },
                ....
                });

With the unnecessary focus callback out of the way, the auto-complete widget will no longer intrude on keystrokes and backspace key. This has been submitted as feedback as well. I hope that it gets addressed one way or another.
